## ゴール

3歳児が大きな色ボタンを押して、中央の丸いパレットで色が気持ちよく変わる様子を楽しめるサイトを作る。学びは「混ぜると色が変わる」を自然に体験させる形で入れ、説明やルール理解を要求しない。

## 体験コンセプト

- プロダクト種別は「教材」より「色あそびのおもちゃ」
- 操作は `色をえらぶ` → `量をえらぶ` → `いれる` の 3 段階に限定
- 主役は中央の混色パレットであり、周辺 UI は補助に徹する
- 失敗状態はほぼ作らず、未選択時のみ短いガイドを出す

## 世界観辞書

- 背景: やわらかい机の上の画用紙。真っ白ではなく温かいオフホワイト
- 盤面: 大きな丸いパレット。少し不揃いで手作り感のある形
- HUD: 最小限の補助表示。文字量を増やさない
- ボタン: ぷっくりした絵の具の塊。押せることが一目で分かる
- 文言: ひらがな中心、短くやさしく、命令口調を避ける
- エフェクト: `ぽたっ` `にゅるっ` `ふわっ` を基調に、速すぎる動きは避ける

## 画面構成

### 全体レイアウト

- 上部: タイトル `いろを まぜよう` と `さいしょから` ボタン
- 中央: 画面で最も大きい混色パレット
- 下部: 12色の絵の具グリッド、量切り替え、`いれる` ボタン、現在色の補助表示
- 補助領域: 投入履歴を小さく表示し、3 色以上混ぜた流れを見失いにくくする

### アンカー条件

- パレットは常に画面中心をアンカーにし、最優先で可視領域を確保する
- 操作系はパレット下端との距離を固定し、画面比率ではなくパレット基準で配置する
- `さいしょから` は誤タップ防止のため右上固定で、パレットから十分離す

## 操作フロー

1. 色を選ぶ
2. `ふつう` または `たっぷり` を選ぶ
3. `いれる` を押す
4. パレット色が更新される
5. 別の色を追加して 3 色以上へ進める
6. `さいしょから` で初期状態に戻す

## 混色ルール

- 基本ルールは「投入済みの顔料総量」に「新しく追加する色」の顔料を足し込む累積混色
- `ふつう = 重み 1`、`たっぷり = 重み 2`
- 最初の 2 色以降も同じ重みルールで累積できる
- 単純な RGB 平均ではなく、絵の具感を出すために減法混色寄りの近似ロジックを使う
- 結果色名は厳密な色名ではなく、近い呼び名をやさしく返す
- 赤や青を多く入れたあとに反対色を 1 回だけ足した場合は、見た目が完全に中間色へ寄るまで急に `むらさきっぽい` へ飛ばさず、優勢色を残して読む
  - 例: `むらさきっぽい`
  - 例: `みどりっぽい`

### 代表レシピ

| レシピ | 想定ラベル | 補足 |
| --- | --- | --- |
| `あか(ふつう) + あお(ふつう)` | `むらさきっぽい` | 同量なら中間色名へ寄せる |
| `あか(たっぷり) + あか(ふつう) + あお(ふつう)` | `あかっぽい` | 優勢な赤が残る間は赤系の呼び名を優先する |
| `あお(たっぷり) + あお(ふつう) + あか(ふつう)` | `あおっぽい` | 優勢な青が残る間は青系の呼び名を優先する |
| `くろ(ふつう) + しろ(ふつう)` | `はいいろっぽい` | 無彩色は暖色名へ寄せない |

## ビジュアル方針

### 配色

- 背景: 画用紙っぽいオフホワイト
- UI 文字/線: 純黒ではなく柔らかいダークグレー
- 基本色: あか、もも、だいだい、きいろ、きみどり、みどり、みずいろ、あお、むらさき、ちゃいろ、しろ、くろ
- 白は枠線や影で背景と分離する

### タイポグラフィ

- 丸みのある日本語ゴシックを使う
- ひらがな中心
- ラベルは大きく短く、補助文も短文で統一

### モーション

- 色投入時にしずくが落ちる
- パレット上で色が広がる
- `たっぷり` は変化量もアニメーション量もやや大きい
- 初期状態では色ボタンが控えめに跳ねて誘導する

## Gemini 相談結果の採否

### 採用

- `さわれる・ぷにぷに・えのぐ感` を主題にした tactile なビジュアル
- ドラッグよりタップ中心の操作設計
- 背景を真っ白ではなく温かいオフホワイトにする方針
- 丸みのある太めの日本語フォント採用
- エラー状態を極小化し、楽しく反応する空状態を用意する方針
- 12色は固定グリッドで一覧表示し、スマホは `3列 x 4行`、タブレット縦は `4列 x 3行` を基準にする
- タブレット横では左右2カラム化を許可し、左にパレット、右に色箱と操作を置く
- 色選択を「絵の具箱」の塊として見せるため、色グリッド全体をひとつのコンテナにまとめる
- 選択状態は枠線だけでなく拡大でも見せる

### 不採用または調整

- 色タップ後に量選択ポップアップを出す案
  - 理由: 今回は `ふつう / たっぷり` を常時見せた方が、3歳児にも保護者にも状態が分かりやすい
- 「どの端末でもスクロール不要を絶対条件」にする案
  - 理由: 12色対応後は端末高によっては無理があるため、今回は「主要操作がすぐ見つかる」ことを優先する

## 受け入れ条件

- 標準12色が一覧から選べる
- 2 色を `ふつう / たっぷり` の 2 択で混ぜられる
- 混ぜた色にさらに別色を足せる
- 主要な操作領域が大きく、3歳児向けに迷いにくい
- 色変化が視覚的に気持ちよい
- GitHub Pages にそのまま公開できる
- スマホ縦・タブレット縦・タブレット横で重なりや極端な窮屈さがない

## 非対象

- 色コード入力
- 細かなスライダー調整
- ログイン、保存、共有
- お絵描きキャンバス
- 色理論の詳細説明

## リスクと対策

| リスク | 対策 |
| --- | --- |
| 単純平均では絵の具っぽい色にならない | 減法混色寄りの近似関数を用意し、テストで期待傾向を固定する |
| 3歳児向けに対して情報量が多くなる | 主役を中央パレットに固定し、文字と操作数を絞る |
| GitHub Pages でパス不整合が起きる | Vite の `base` を公開先前提で設定する |
| ボタンが小さく誤タップが増える | 最小タップ領域と gap を先に設計値として固定する |

## 性能目標

- 初回表示は軽量に保つ
- 主要操作は 1 タップで進行できる
- アニメーションは短く、待ち時間を感じさせない

## 実装境界

- フロントエンドは `Vite + React + TypeScript`
- UI は 1 ページ構成
- 公開は GitHub Pages
- 依存は最小限に保ち、視覚演出は CSS と少量の JavaScript で実現する
